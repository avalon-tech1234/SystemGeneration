
Алгоритм нормализации системы.

1. Цель.
Получить из сгенерированной системы уравнений (состоит из небольшого количества относительно сложных уравнений)
нормализованную систему уравнений без потери информации.
Нормализованной системой называем систему, каждое уравнение которой имеет вид (в порядке убывания приоритета):
x + yz = 0,
x + y + z = 0 или
x + y + с = 0 // может появиться только в конце обработки уравнения
Где x, y, z - переменные, а с - константа.

2. Задача.
Имеется система из n уравнений второй степени, содержащая n переменных ( x_0, x_1, ..., x_(n-1) ).
Требуется ее нормализовать.

3. Идея.
Действуем с помощью преобразования имеющихся уравнений (уравнения ядра) к шаблонному виду
путём введения новых переменных и добавления новых уравнений(уравнения связи),
связывающих вводимые переменные с имеющимися.

4. Псевдокод

Полином Lr - текущий левый фрагмент -
	равен части текущего полинома от левого края до метки (указывает на первый нерассмотренный моном).
	Изначально метка указывает на первый моном первого уравнения, Lr = 0.
Число z - номер переменной-кандидата на добавление. Изначально равно n.
	При добавлении новой переменной увеличивается на 1.

Для каждого уравнения ядра проделать следующее:
	Пока уравнение не нормализовано (пока метка не указывает на 1 или на "после последнего монома"):
		Сдвинуть метку на 1 вправо
		Если полином P = x_z + Lr - шаблонный:
			Вводим новую переменную x_z
			Добавляем новое уравнение связи x_z + Lr = 0
			Пробежать по всем уравнениям
				Если Lr присутствует как слагаемое в левой части
					Заменить Lr на x_z (и сразу упростить)
			z++
